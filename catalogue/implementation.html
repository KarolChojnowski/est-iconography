Intentional scale

Use explicit sizes

Without a size class, the icon uses 1em. The utilities below cover the normal UI and Icon display sizes.

{% assign sizes = '16,24,32,48,64' | split: ',' %} {% for size in sizes %}

For an exceptional size, set --est-icon-size on the SVG. Review UI icons at 16px and Icons at 48px before introducing scaled variants.

Delivery contract

Keep the generated paths together

Complete release

Copy the versioned release directory into an application-owned public asset location.

Selected bundle

Use the catalogue selection or CLI; the subset retains the same sprite, SVG, stylesheet and manifest paths.

Same-origin sprites

Serve external sprites from the same application origin and preserve the generated symbol IDs.

Versioned caching

Version or fingerprint the containing asset directory when products use long-lived caching.

Before release

Implementation checklist

  • Use a canonical sprite ID or generated individual SVG path.
  • Load the optional helper stylesheet once, not per component.
  • Choose and review an intentional rendered size.
  • Set colour through color, not path fills.
  • Keep decorative and control-owned SVGs hidden from assistive technology.
  • Put accessible names and focus behaviour on the containing control.
  • Retain the bundle licence notices in the consuming product.

For the complete accessibility decision model, read Accessible icon use.
